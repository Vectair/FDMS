ADSB monitoring - Storing ICAO hex codes with the appropriate airframe in the registration database can allow users to visually look up aircraft preferably on an internatl ADSB map but external is acceptable, for a rough location of the aircraft.
Multiple users on the same itteration - Air Traffic, Flying Club and Operations all have access to the same instance of Flite so the flying club can add their own sorties to the board, as well as view the current schedule for self deconfliction. Can also be used for stat tracking when ATC are off. Operations can use this for monitoring movements for escort and billing as well as the booking section for adding flights to the calendar and liveboard. Each version of this could have restrictions built in for editor vs viewer vs null etc.
SQLite Database - Move all databases to SQLite for: fast indexed search by registration, stable pagination, cleaner update/delete/edit handling, proper audit/history storage, fewer giant CSV merge problems, easier backup/export/import, avoiding huge DOM/data rebuilds, future multi-table relationships.
